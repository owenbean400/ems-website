# Sync Rescue record

## POST `/v1/ems/rescue/sync/{rescue_id}`

Register a rescue as being synced to the latest from the device. When updating a rescue record on a device based on the cloud data, send this HTTP request to let the backend know the record is in sync with the device.

### Request

#### Headers

```
"Authorization": "Bearer <Token>"
```

| Path Parameters | |
| ---- | ----- |
| rescue_id | The ID of the rescue. |

#### Body
```
{
    "device_id": [string] UUID device id
}
```

### Response `200`

Rescue record has been set as sync up for the device.

#### Body
```
{
    "device_id": [string] UUID device id
}
```

### Response `201`

Rescue record has made a new tracking record of the sync up for the device.

#### Body
```
{
    "device_id": [string] UUID device id
}
```

### Response `401`

Unauthorizated for the user from bearer token.

#### Body
```
{
    "Error": "Unauthorized",
    "message": [string] Authorization issue
}
```

#### Messages

- You are not authorized to access this resource
> The token from Firebase is invalid
- Authorization header is missing
> There is no "Authorization" in the header request.
- Invalid authorization header format
> The "Authorization" header does not have "Bearer Token" format.


### Response `404`

The device ID or rescue record could was not found.

#### Body
```
{}
```

### Response `500`

Internal error with the server.

#### Body
```
{
    "error": "Internal Server Error",
    "message": "Internal Server Error"
}
```
