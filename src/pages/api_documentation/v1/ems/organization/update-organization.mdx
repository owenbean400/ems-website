# Update Organization

## PATCH `/v1/ems/organization/{org_id}`

Updates organization by the owner. Can update either name, owner of organization, or both.

### Request

Headers

```
"Authorization": "Bearer <Token>"
```

| Path Parameters | |
| --------------- | - |
| org_id | The Organization ID |

Body
```
{
    "name": [string] (Optional) The name of organization to update to,
    "user_id": [number] (Optional) The new owner ID to update to
}
```

### Response `200`

Update has been completed.

### Response `401`

Authorization issue.

#### Body
```
{
    "Error": "Unauthorized",
    "message": [string] Authorization issue
}
```

#### Messages

- You are not authorized to access this resource
> The token from Firebase is invalid
- Authorization header is missing
> There is no "Authorization" in the header request.
- Invalid authorization header format
> The "Authorization" header does not have "Bearer Token" format.

### Response `500`

Internal error with the server. This includes issues with the data for updating the record.

#### Body
```
{
    "error": "Internal Server Error",
    "message": "Internal Server Error"
}
```
